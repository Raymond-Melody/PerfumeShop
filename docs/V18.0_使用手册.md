## 1. V18 新功能概览

V18.0 是 PerfumeShop 的"全栈现代化"版本，聚焦四大方向：

| 方向 | 核心功能 | 面向用户 |
|------|---------|---------|
| **移动端优先** | 底部 Tab Bar、移动触控优化、一键支付 | 全平台访客 |
| **PWA 3.0** | 离线购物车、推送通知、Background Sync | 移动端访客 |
| **架构现代化** | API 认证与限流、Web Components、性能索引 | 开发者/运维 |
| **监控增强** | 业务指标采集、转化漏斗、健康检查增强 | 运营/管理员 |

## 2. 移动端使用指南

### 2.1 底部 Tab Bar 导航

在手机浏览器（≤768px 宽度）访问网站时，页面底部会显示固定导航栏：

```
┌──────────────────────────────────────────────┐
│                                              │
│               页面内容区域                     │
│                                              │
├──────────────────────────────────────────────┤
│  🏠 首页   📂 分类   🛒 购物车   👤 我的    │
└──────────────────────────────────────────────┘
```

**操作说明：**
- 点击图标切换对应页面
- 当前页面图标高亮显示
- 购物车图标显示商品数量角标
- 适配底部安全区域（iPhone X 及以上机型）

### 2.2 移动触控优化

- **按钮尺寸**：所有可点击元素最小 44px×44px，符合 iOS/Android 触控规范
- **输入框**：表单输入框字号 ≥16px，防止 iOS 自动缩放
- **日期选择**：使用浏览器原生日期选择器（`type="date"`）
- **地址选择**：省市区三级联动，滚动选择友好

### 2.3 移动端支付

**操作步骤：**
1. 在结算页选择支付方式（银行卡/微信支付/支付宝）
2. 勾选"记住支付方式"（下次自动填充）
3. 点击大按钮确认支付
4. 跳转对应支付页面完成支付

> ⚠️ 微信支付和支付宝需在 `includes/payment_config.asp` 中填入真实商户凭证后启用。

## 3. PWA 离线与推送通知

### 3.1 离线购物车

当网络断开时，购物车操作（添加/删除/修改）会自动进入离线队列：

```
正常模式: 点击"加入购物车" → 实时提交 → 正常响应
离线模式: 点击"加入购物车" → IndexedDB 暂存 → 恢复网络后自动同步
```

**离线购物车查看：** 访问 `/offline.html` 查看离线缓存内容：
- 已缓存的页面预览
- 待同步的离线购物车商品列表
- 上次在线时间
- 一键重试连接

### 3.2 推送通知订阅

**首次使用流程：**
1. 访问支持推送的页面，浏览器弹出通知权限请求
2. 点击"允许"后，系统自动订阅推送服务
3. 后续运营可通过后台发送推送通知

**手动管理：**
- 点击页面的"通知设置"按钮可随时开启/关闭推送
- 在浏览器设置中可撤销通知权限

**推送通知效果：**
- 订单状态变更提醒（已发货、已签收）
- 限时秒杀开始提醒
- 活动促销推送
- 点击通知直接跳转对应页面

> ⚠️ 推送通知需要 HTTPS 环境，本地 HTTP 测试不可用。

### 3.3 SSE 实时通知

网站顶部通知图标会实时显示未读通知数：

| 通知类型 | 触发条件 | 图标 |
|---------|---------|------|
| 订单通知 | 订单状态变更 | 📦 |
| 系统通知 | 系统公告 | 🔔 |
| 活动通知 | 秒杀/拼团开始 | 🎉 |
| 社区通知 | 评论/点赞/关注 | 💬 |

## 4. 前端 Web Components

V18.0 引入了 3 个 Web Components（自定义 HTML 元素），无需任何 JavaScript 框架即可使用：

### 4.1 产品卡片 `<product-card>`

**用法：**
```html
<product-card 
  data-id="123"
  data-name="晨露玫瑰淡香水"
  data-price="299"
  data-image="/images/products/rose.jpg"
  data-category="花香调"
  data-rating="4.5"
  data-badge="新品"
  data-link="/product.asp?id=123"
  data-lazy="true">
</product-card>
```

**特性：**
- 图片懒加载（IntersectionObserver）
- Hover 显示详情浮层
- 星级评分自动渲染
- 类型徽章（定制/限定/新品）

### 4.2 星级评分 `<review-stars>`

**只读模式（显示评分）：**
```html
<review-stars data-value="4.5" data-readonly="true"></review-stars>
```

**交互模式（用户评分）：**
```html
<review-stars data-value="0" data-name="rating" id="reviewStars"></review-stars>
```

**键盘操作：**
| 操作 | 效果 |
|------|------|
| 鼠标点击 | 选择星级 |
| Alt + 点击 | 选择半星 |
| Hover | 预览评分效果 |

**事件监听：**
```javascript
document.querySelector('#reviewStars').addEventListener('change', function(e) {
  console.log('评分为:', e.detail.value); // 如 4.5
});
```

### 4.3 搜索自动补全 `<search-autocomplete>`

**用法：**
```html
<search-autocomplete 
  data-api="/api/search_suggestions.asp"
  data-placeholder="搜索香水、香调..."
  data-min-length="2">
</search-autocomplete>
```

**操作说明：**
| 操作 | 效果 |
|------|------|
| 输入 ≥2 个字符 | 自动弹出建议列表（350ms 防抖） |
| ↑ ↓ 方向键 | 在建议项中移动 |
| Enter | 跳转至选中产品 |
| Esc | 关闭建议列表 |
| 点击"查看全部结果" | 搜索当前输入关键词 |

## 5. API 认证与限流（开发者）

### 5.1 认证模式

V18 支持两种 API 认证模式，由 `includes/config.asp` 中的 `FEATURE_API_AUTH` 控制：

**模式一：Session 认证（前台页面调用）**
- 用户登录后自动携带 Session
- API 自动识别用户身份
- 无需额外操作

**模式二：API Key + HMAC 签名（第三方应用/服务端调用）**

**调用示例：**
```vbscript
' 服务端生成签名
Dim apiKey, timestamp, signStr, signature
apiKey = "YOUR_API_KEY"
timestamp = Int(DateDiff("s", "1970-01-01 00:00:00", Now()))
signStr = apiKey & timestamp & API_AUTH_SECRET
signature = SHA256Hash(signStr)
```

**HTTP 请求头：**
```
X-API-Key: YOUR_API_KEY
X-API-Signature: 生成的HMAC签名
X-API-Timestamp: 当前Unix时间戳
```

### 5.2 速率限制

默认每个客户端 60 秒内最多 60 次 API 请求：

**响应状态：**
- 正常：HTTP 200 + JSON 数据
- 超限：HTTP 429 + JSON 错误 + `Retry-After` 头

**响应头示例：**
```
HTTP/1.1 429 Too Many Requests
Retry-After: 60
Content-Type: application/json

{"error":"Rate limit exceeded","code":"RATE_LIMITED","retry_after":60}
```

**配置调整：**
在 `includes/rate_limiter.asp` 中修改：
```vbscript
Const RL_DEFAULT_MAX = 60    ' 每窗口最大请求数
Const RL_DEFAULT_WINDOW = 60 ' 时间窗口（秒）
```

| 端点 | 认证 | 限流 | 说明 |
|------|------|------|------|

## 7. 后台运营功能

### 7.1 秒杀活动管理

路径：`/admin/operation/flash_sale.asp`

**操作流程：**
1. 新增秒杀活动 → 选择商品、设置秒杀价、开始/结束时间
2. 设置库存上限（每人限购数量）
3. 活动开始后，前台 `/flash_sale.asp` 自动展示倒计时
4. 库存售罄自动关闭

### 7.2 拼团活动管理

路径：`/admin/operation/group_buy.asp`

**操作流程：**
1. 新增拼团活动 → 选择商品、设置拼团价、参团人数
2. 用户在前台 `/group_buy.asp` 发起或参与拼团
3. 达到参团人数后自动成团
4. 未成团自动退款

### 7.3 数据质量检查

路径：`/admin/system/data_quality.asp`

自动执行 11 项数据质量检查并生成评分：

| 检查项 | 说明 |

| 产品价格 | 检查价格为 0 或负数的异常记录 |
| 库存数量 | 检查库存为负数的异常记录 |
| 产品名称 | 检查名称为空的产品 |

## 8. 数据库与缓存管理

### 8.1 性能索引清单

V18 新增 13 个性能索引（位于 `database/v18_perf_indexes.sql`）：

| 表名 | 索引键 | 优化场景 |
|------|-------|---------|
| Products | ProductType + IsActive | 产品列表按类型筛选 |
| Products | Category + IsActive | 产品列表按分类筛选 |
| Products | CreatedAt + IsActive | 新品排序 |
| Products | ProductName | 产品搜索 |
| Orders | UserID + CreatedAt | 用户订单历史 |
| Orders | Status + CreatedAt | 订单管理按状态筛选 |
| OrderItems | OrderID + ProductID | 订单明细查询 |
| AppLogs | CreatedAt + LogType | 日志按类型查询 |
| UserBehavior | UserID + BehaviorType + CreatedAt | 用户行为分析 |

### 8.2 索引维护

```sql
-- 重建索引（建议每月执行）
ALTER INDEX ALL ON Products REBUILD WITH (ONLINE=ON);
ALTER INDEX ALL ON Orders REBUILD WITH (ONLINE=ON);

-- 更新统计信息（建议每周执行）
UPDATE STATISTICS Products WITH FULLSCAN;
UPDATE STATISTICS Orders WITH FULLSCAN;
```

### 8.3 缓存清理

| 场景 | 操作 |
|------|------|
| 产品新增/修改 | 系统自动调用 `CM_ClearProductCaches()` |
| 全站缓存清理 | 重启 IIS / 回收 Application Pool |
| 手动清理 | 在代码中调用 `Application.Contents.Remove("CACHE_*")` |

## 10. 故障排除

### 10.1 API 返回 429 Too Many Requests

**原因：** 请求超过速率限制（默认 60 次/60 秒）
**解决：**
- 等待 `Retry-After` 头指定的时间后重试
- 在 `rate_limiter.asp` 中增大 `RL_DEFAULT_MAX`

### 10.2 API 返回 401 Unauthorized

**原因：** API 认证失败
**解决：**
- 确认 `X-API-Key` 和 `X-API-Signature` 请求头正确
- 确认签名时间戳在 ±300 秒范围内
- 确认 `FEATURE_API_AUTH` 配置正确

### 10.3 推送通知未显示

**原因排查：**
1. 确认网站使用 HTTPS（推送通知要求安全上下文）
2. 检查浏览器通知权限 → 设置为"允许"
3. 确认 VAPID 公钥配置正确
4. 确认 `sw.js` 已注册且激活

### 10.4 离线购物车未同步

**解决步骤：**
1. 确认网络已恢复
2. 刷新页面触发 Background Sync
3. 或手动访问 `/offline.html` 检查队列状态
4. 如仍未同步，检查 `sw.js` 中 `sync-offline-cart` 事件处理

### 10.5 移动端样式异常

**解决：**
- 确认 `css/mobile-first.css` 和 `css/responsive.css` 已加载
- 确认 viewport meta 标签存在：`<meta name="viewport" content="width=device-width, initial-scale=1">`
- 尝试清除浏览器缓存后刷新

## 11. 常见问题 FAQ

### Q: V18 需要更新浏览器吗？
A: Web Components 需要 Chrome ≥67、Firefox ≥63、Safari ≥10.1、Edge ≥79。建议使用最新版浏览器。

### Q: 如何测试 API 认证？
A: 运行 `test_v18_smoke.ps1` 脚本查看 API 认证和限流是否正常工作。

### Q: 移动端底部 Tab Bar 在平板显示吗？
A: 仅在 ≤768px 宽度的设备显示。平板和桌面端不显示底部导航。

### Q: 秒杀活动和普通购物车冲突吗？
A: 不冲突。秒杀商品通过独立库存管理，与购物车共享结算流程。

### Q: 如何关闭推送通知？
A: 点击页面"通知设置"按钮取消订阅，或在浏览器地址栏左侧点击🔒图标 → 通知 → 阻止。

### Q: V18 数据库脚本能否重复执行？
A: 可以。所有脚本包含 `IF NOT EXISTS` 幂等性检查，重复执行不会报错。

### Q: 性能索引的 ONLINE=ON 是什么？
A: 允许在不停机的情况下创建/重建索引。需要 SQL Server Enterprise Edition。Standard Edition 需移除 ONLINE 选项。

### Q: Web Components 不显示怎么办？
A: 确认 `includes/footer.asp` 中包含以下脚本引用：
```html
<script src="/js/components/product-card.js?v=18.0" defer></script>
<script src="/js/components/review-stars.js?v=18.0" defer></script>
<script src="/js/components/search-autocomplete.js?v=18.0" defer></script>
```
