## 变更日志

### P0 - 紧急项（安全与基础设施）

#### SQL 注入全面加固
- **`admin/login.asp`**: 使用 `DAL_Users_AdminLogin()` 参数化查询替代字符串拼接
- **`admin/includes/auth.asp`**: 管理员身份验证迁移到 `DAL_GetRow()` 参数化查询
- **`user/login.asp`**: 登录验证使用 `DAL_Users_GetByUsername()` 参数化
- **`products.asp`**: 搜索查询使用 `DAL_GetList()` 参数化，消除 `SafeSQL()` 依赖
- **`includes/password_utils.asp`**: 密码更新使用 `DAL_Execute()` 参数化
- **`includes/audit_utils.asp`**: 完整审计日志模块参数化迁移（`DAL_Execute`/`DAL_GetScalar`）
- **`user/addresses.asp`**: 全部 CRUD 操作迁移到 `DAL_Users_*Address*` 系列函数
- **`checkout_address_handler.asp`**: 地址保存使用 `DAL_Users_AddAddress()` 参数化
- **`api/get_areas.asp`**: 地区查询使用 `DAL_GetRow()`/`DAL_GetList()` 参数化
- **`cart.asp`**: 购物车操作迁移到 `DAL_Execute()` 参数化
- **`includes/email_utils.asp`**: 邮件查询使用 `DAL_GetScalar()` 参数化
- **`user/forgot.asp`**: 密码找回查询参数化
- **`user/settings.asp`**: 用户设置操作参数化
- 总计修复 **14+** 个页面的 SQL 注入风险点

#### SQL Server 连接稳定性
- **`includes/connection.asp`**: 完整重写
  - 连接池健康检查：`OpenConnection()` 中执行 `SELECT 1` 验证
  - 自动重试机制：最多 3 次重试，500ms 间隔
  - MSOLEDBSQL 自动回退：主驱动失败时自动切回 SQLOLEDB
  - 心跳检测日志：按概率（1/50）记录 `AppLogs` 心跳
  - 增强错误日志：断开原因记录到 `Session("DBLastError")`

#### DAL 覆盖率提升 (~30% → ~70%)
- **新建 DAL 模块**：
  - `includes/dal_admin.asp` (250行) - 管理后台查询
  - `includes/dal_cart.asp` (170行) - 购物车操作
  - `includes/dal_checkout.asp` (215行) - 结算流程
  - `includes/dal_finance.asp` (266行) - 财务相关
  - `includes/dal_inventory.asp` (302行) - 库存管理
- **增强现有 DAL**：
  - `includes/dal_users.asp` (+94行) - 新增地址管理函数
  - `includes/dal_products.asp` (+45行) - 搜索建议、搜索历史
- **迁移关键页面**：
  - `api/batch_operations.asp` → 全部使用 `DAL_Execute()` 参数化
  - `checkout_utils.asp` → `GetAreaNameById` 使用 `DAL_GetList()`
  - `checkout_cart_loader.asp` → 3 个查询迁移到 DAL 函数

### P1 - 重要项（安全增强与功能完善）

#### CSP 安全加固
- **`includes/config.asp`**: CSP 头 `script-src` 移除 `'unsafe-inline'`
- **`includes/header.asp`**: 所有 `<script>` 和 `<style>` 标签使用 `Session("csp_nonce")`
- **`api_response.asp`**: API 响应中 CSP 安全隔离

#### CSRF 稳定性增强
- **`includes/connection.asp`**: `GenerateCSRFToken()` 添加历史 token 池（最大5个）
- **`ValidateCSRFToken()`**: 支持主 token + 历史池双重验证
- **API 端点**: 统一使用 `API_CheckCSRF()`

#### 搜索功能完善
- **`includes/dal_products.asp`**: 新增 `DAL_Products_GetSuggestions()`、`DAL_Products_GetSearchHistory()`、`DAL_Products_RecordSearch()`
- **`api/search_suggestions.asp`** *(新建)*: 搜索建议 API（type=suggest / history / clear_history）
- **`products.asp`**: 搜索历史记录 + 加权排序（名称前缀 > 名称包含 > 描述匹配）
- **`includes/header.asp`**: 搜索建议前端 UI（键盘导航、自动补全）

#### 财务报表增强
- **`admin/finance/reports.asp`**: 多维度报表增强

#### 错误处理统一
- **`error.asp`**: 完整重写，支持错误码 403/404/500/503/400/429

  - 渐变 UI 主题 + 响应式设计
  - 返回上页 / 返回首页按钮

## 升级注意事项

1. **配置步骤**: 将 `web.config` 中数据库连接字符串更新为 MSOLEDBSQL 格式
2. **依赖安装**: 如需 MSOLEDBSQL 驱动，运行 `database/enable_tcpip_sqlserver.ps1`
3. **备份还原**: 使用备份文件 `V16_Full_for_V17_Upgrade` 进行还原
4. **缓存清理**: 升级后建议重启 IIS 或重置 Application Pool 以清理缓存
5. **代码规范**: 所有新查询必须使用 `DAL_Execute()` / `DAL_GetList()` 等参数化函数，禁止字符串拼接
