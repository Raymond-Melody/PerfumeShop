# V16 版本迭代升级计划

## 版本定位
基于 V15 的安全基础架构，V16 聚焦于 **功能完善和体验优化**，将 V15 中预留但未启用的能力逐项激活，同时提升用户体验和运营效率。

---

## 一、V15 遗留项（P0-P1 激活）

### Task 1: 安全升级收尾（P0 → 启用）
- [ ] **1.1 启用 FEATURE_PASSWORD_V3** — SHA-512 + 1000轮迭代 + 新 salt
  - 影响文件: `config.asp`, `password_utils.asp`, `user/login.asp`, `user/register.asp`
  - 兼容策略: 登录时先尝试V3验证，失败则回退V2，成功后自动升级哈希
  - 风险: 低（password_utils.asp 已就绪，只差 feature flag）

- [ ] **1.2 启用 FEATURE_DAL_ENABLED** — 统一数据访问层
  - 影响文件: `config.asp`, 全站所有数据访问页面
  - 策略: 渐进式迁移，先迁移 user/ 和 api/ 目录（约26个文件）
  - 目标: V16 结束时 DAL 覆盖率 ≥ 50%（约130个文件）
  - 优先级页面: user/login.asp, user/register.asp, api/*, products.asp, product.asp

- [ ] **1.3 安装 MSOLEDBSQL 驱动** — 数据库驱动升级
  - 下载安装: https://aka.ms/downloadmsoledbsql
  - 启用 `FEATURE_MSOLEDBSQL = True` 后测试全站
  - 自动回退机制已就绪（connection.asp 第53-64行）

### Task 2: API 标准化（P1）
- [ ] **2.1 启用 FEATURE_API_V1** — 统一 API 响应格式
- [ ] **2.2 升级 cart_add.asp** — 从 HTML alert 改为 JSON 响应
  - 当前: `<script>alert('...')</script>` 
  - 目标: `{"code":200/400/403,"message":"...","data":{...}}`
- [ ] **2.3 API 文档化** — 生成所有14个API端点的使用文档

## 六、技术风险与对策

| 风险 | 影响 | 对策 |
|------|------|------|
| MSOLEDBSQL 驱动兼容性 | 数据库连接失败 | 自动回退 SQLOLEDB（已实现） |
| Password V3 升级 | 老用户无法登录 | V3→V2 回退 + 自动升级策略 |
| DAL 迁移覆盖不全 | 部分页面未迁移 | 优先级排序，高频页面优先 |
| Email 发送阻塞页面 | 用户体验下降 | 邮件队列 + 异步处理 |
| 第三方 CDN 不可用 | JS/CSS 加载失败 | 本地 fallback |
