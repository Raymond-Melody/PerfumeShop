# V18 ↔ V19「管理中心」全模块对比分析报告

> 对象：香氛定制电商系统 · 管理中心（Admin）
> 版本：V18（Classic ASP）↔ V19（ASP.NET Core 8 · Blazor Server · MudBlazor）
> 说明：本报告为「工作安排」7 项要求的整合交付物（第 6 项）。第 7 项「可执行计划」已制定并**全部执行落地**，其结果与冒烟验证一并收录于第 6、7 章。

---

## 1. 概述

### 1.1 技术栈对照
| 维度 | V18 | V19 |
|---|---|---|
| 运行时 | Classic ASP（VBScript）+ IIS | ASP.NET Core 8 + Kestrel（端口 5207） |
| 渲染 | 服务端拼接 HTML | Blazor Server（InteractiveServer 电路）+ MudBlazor |
| 数据访问 | ADODB + 手写 SQL（`includes/dal*.asp`） | EF Core（`PerfumeShopContext`）+ 仓储层 |
| 认证 | Session（`admin/includes/auth.asp`） | Cookie 认证 + Claims（`/login`，`Login.razor`） |
| 授权 | `admin/includes/role_auth.asp`（257 行，按角色码判断）+ 各模块 `includes/auth.asp` | 数据驱动 RBAC：`AdminRole.ModuleAccess` + `RolePermission` + 授权策略 `Module.{code}` |
| 导航 | 每模块各自 `includes/nav.asp` | 单一 `MainLayout.razor` 统一侧边栏 + 门禁 |
| 审计 | `AdminLog`/`AppLog` 分散写入 | `IAdminAuditor` 统一写 `AdminAuditLog`（操作人/动作/目标/详情） |

### 1.2 对比范围
V18 与 V19 的管理中心均由 **10 个模块**构成，均已纳入对比：

系统管理、财务、采购、物流、库存、运营、生产中心、半成品中心、技术中心、数据分析。

### 1.3 方法与证据
- V18 侧：清点 `admin/**/*.asp`（系统 21 / 财务 24 / 采购 36 / 运营 29 / 生产 13 / 半成品 10 / 技术中心 17 / 物流 8 / 库存 3 / 分析 2 页）。
- V19 侧：以认证态全路由巡检得到的 **128 条 `@page` 路由**为准（源码位于 `src/PerfumeShop.Admin/Components/Pages/**`）。
- 全文以「具体文件路径」为对比依据（见第 9 章证据索引）。

### 1.4 结论摘要
- **功能覆盖**：V19 已**对齐 V18 全部 10 个模块**的主功能，并在系统安全域实现**超越**（运行时 IP 拦截、数据驱动 RBAC、统一审计、登录风控）。
- **实现方式**：由「服务端拼接 + Session + 分散鉴权」升级为「组件化 + Claims + 数据驱动策略 + 单点门禁」。
- **新增/优化**：统一审计埋点覆盖全模块写操作；门户按权限显隐；共享密码散列（全角/半角 + V1/V2/V3 自动升级）；共享样式基线。
- **移除/简化**：V18 部分「建表/自检」类运维脚本（如 `create_formula_tables.asp`）在 V19 以迁移/初始化方式替代，不再作为页面暴露。
- **本次交付还修复 3 个严重缺陷**（详见第 7 章）：全站 500（IP 中间件缓存 Size）、技术中心全模块 302 锁死（角色名/码不匹配）、香调/瓶型/基香保存报错（EF keyless 误配）。

### 3.2 实现方式差异
- **鉴权**：V18 每个系统页顶部 `<!--#include file="includes/auth.asp"-->` + `role_auth.asp` 按角色码字符串判断；V19 用 `@attribute [Authorize(Policy = "Module.system")]` + `ModuleAccessHandler`（从 `context.User` 取 `RoleCode` claim，SUPER_ADMIN 直通，其余查 `AdminRole.ModuleAccess`）。
- **审计写入**：V18 分散调用；V19 统一 `IAdminAuditor.LogAsync(adminId, adminName, action, targetType, targetId, targetName, details)`，隔离 `PerfumeShopContext` 写 `AdminAuditLog`，操作人取自 `AuthenticationState`（交互态无 `HttpContext`，故显式传参）。
- **登录风控**：V19 `ILoginSecurityService`（60s 令牌桶 + 5 次失败自动封 IP 7 天，写 `AdminLog`+`LoginAlert`），对齐并强化 V18 的 `login_monitor` 语义。
- **密码兼容**：V19 抽取 `PerfumeShop.Shared/Security/PasswordHasher`（pepper `P3rfum3Sh0p_S@lt_2026!`，全角/半角标准化，V1/V2/V3 校验 + 登录时自动升级到 V3），完整继承 V18 `config.asp` 口令语义。

### 3.3 新增 / 优化 / 移除
- **新增**：运行时 IP 拦截中间件；数据驱动 RBAC 策略 + 单点门禁（`MainLayout.razor` 的 `CheckAccess`/`ModuleCodeFor`）；门户按权限显隐卡片（`Home.razor`）；`SeedData` 开发辅助页；备份下载端点。
- **优化**：管理员增删改由多页合一；角色权限从字符串判断升级为可视化 CRUD 矩阵；安全监控 Blazor 原生重建。
- **移除/简化**：`V8_USER_MANUAL.asp`（1014 行静态手册）在 V19 精简为 `Guide.razor`；`statistics.asp` 并入分析/门户。

**冒烟过程中发现并修复的 3 个严重缺陷**：

| # | 缺陷 | 影响 | 修复 |
