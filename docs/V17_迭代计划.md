# V17 版本迭代升级计划

## 版本定位
基于 V16 的功能完善基础，V17 聚焦于 **安全加固 + 稳定性提升 + 用户体验打磨 + 运营效率**，将 V16 遗留的 MSOLEDBSQL 和 DAL 覆盖率补全，同时系统性地提升代码质量和运维能力。

---

## 一、V16 遗留项收尾 (Week 1-2)

### Task 1: MSOLEDBSQL 驱动激活 (P0)
- [ ] **1.1 配置 SQL Server TCP/IP 协议**
  - SQL Server 配置管理器 → YOURPERFUME 协议 → 启用 TCP/IP
  - 设置固定端口 1433（或动态端口）
  - 重启 SQL Server 服务
  - 验证: `sqlcmd -S localhost,1433\YOURPERFUME`

- [ ] **1.2 切换 Feature Flag**
  - `FEATURE_MSOLEDBSQL = True`
  - 全站冒烟测试（先测试核心页面，逐步扩大到全部）
  - 验证回退机制（故意填错连接串测试）
  - 对比 SQLOLEDB vs MSOLEDBSQL 性能基准

### Task 2: DAL 覆盖率提升 (P0)
- [ ] **2.1 高优先级页面迁移**
  - 目标: DAL 覆盖率 30% → 60%
  - 优先迁移（约30个文件）:
    - 用户模块: user/login.asp, user/register.asp, user/orders.asp, user/favorites.asp
    - API模块: api/cart_*.asp, api/order_*.asp, api/favorites.asp
    - 前台核心: products.asp, product.asp, cart.asp, checkout.asp

- [ ] **2.2 DAL 增强**
  - 添加参数化查询支持（防SQL注入）
  - 添加连接池监控
  - 批量操作性能优化

### Task 3: 国际化 (I18N) 激活 (P3 → P2)
- [ ] **3.1 翻译内容填充**
  - locale/zh-CN.asp: 中文基准（从现有代码提取）
  - locale/en-US.asp: 英文翻译
  - 覆盖: 导航、按钮、表单标签、错误提示、邮件模板

- [ ] **3.2 启用 FEATURE_I18N**
  - 语言自动检测（浏览器 Accept-Language）
  - 手动切换（URL参数 ?lang=zh|en 或 Cookie）

## 二、安全加固 (Week 2-3)

### Task 4: SQL注入防护 (P0 - 紧急)
- [ ] **4.1 全站参数化查询审计**
  - 扫描所有 `conn.Execute("SELECT ..." & Request)` 模式
  - 发现漏洞: admin/operation/points.asp (userId未做CLng转换)
  - 批量修复: 所有数值参数强制类型转换，字符串参数使用SafeSQL()

- [ ] **4.2 输入验证增强**
  - 统一前端+后端双重验证
  - 表单字段白名单校验
  - 文件上传类型严格限制

### Task 5: CSRF Token 稳定性 (P1)
- [ ] **5.1 修复Token验证稳定性**
  - 排查: ASP Session在特定场景下丢失CSRFToken
  - 增加Token过期刷新机制
  - 添加AJAX请求自动附带Token

### Task 6: XSS/CSP 加固 (P1)
- [ ] **6.1 CSP策略审查**
  - 收紧 `unsafe-inline` 使用
  - 迁移内联脚本到 nonce-based CSP
  - 添加 `require-trusted-types-for 'script'`

## 三、SQL Server 稳定性 (Week 2)

### Task 7: 数据库运维保障 (P0)
- [ ] **7.1 SQL Server 意外停止排查**
  - 检查 Windows 事件日志 → 找到根因
  - 可能原因: 内存不足、磁盘空间、补丁问题
  - 设置服务自动恢复: 第一次失败→重启服务，第二次→重启计算机

- [ ] **7.2 数据库健康监控**
  - 新增 api/health_check.asp（DB连接+磁盘空间+响应时间）
  - 定时任务: 每5分钟健康检查，异常时记录日志

- [ ] **7.3 备份验证**
  - 自动验证备份文件完整性（RESTORE VERIFYONLY）
  - 备份失败时发送告警

## 四、用户体验打磨 (Week 3-4)

### Task 8: 产品浏览体验 (P1)
- [ ] **8.1 放宽产品浏览权限**
  - 当前: 未登录用户无法浏览 products.asp 和 customize.asp
  - 目标: 未登录可浏览产品列表和详情，仅购买/定制需登录
  - 实现: 移除 products.asp 和 product.asp 的强制登录检查

- [ ] **8.2 产品搜索功能**
  - 新增搜索框（全站头部）
  - 支持: 产品名称、香调、品牌模糊搜索
  - 搜索结果页: 产品卡片网格 + 筛选

- [ ] **8.3 产品图片批量上传**
  - 后台产品管理增加拖拽上传
  - 自动生成缩略图
  - WebP自动转换

### Task 9: 地址管理优化 (P2)
- [ ] **9.1 三级联动增强**
  - 当前 get_areas.asp 数据可能不全
  - 更新省市区数据（基于最新民政部数据）
  - 添加搜索自动补全

- [ ] **9.2 智能地址推荐**
  - 基于历史订单自动填充
  - GPS定位获取当前位置

### Task 10: 购物流程优化 (P2)
- [ ] **10.1 购物车持久化**
  - 当前登录后合并匿名购物车
  - 增强: 跨设备同步（通过UserID关联）

- [ ] **10.2 库存实时校验**
  - 加入购物车时检查库存
  - 结算时再次校验
  - 库存不足友好提示

## 九、技术风险与对策

| 风险 | 影响 | 对策 |
|------|------|------|
| SQL Server反复停止 | 全站不可用 | 事件日志分析根因 + 自动重启策略 |
