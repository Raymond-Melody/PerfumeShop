## 二、V14.5 → V14.6 版本更新说明

### 2.1 V14.5 完成的工作（上一版本）

V14.5 是系统安全加固与代码重构的关键版本，主要完成了以下工作：

- **CSP Nonce 方案**: 每次请求生成唯一随机 nonce，替代 `'unsafe-inline'`，增强内容安全策略
- **HSTS 强化**: 添加 `includeSubDomains` 标志，确保全站 HTTPS 强制传输
- **参数化查询**: 新增 `ExecuteParameterizedQuery()` 函数，使用 ADODB.Command 防止 SQL 注入
- **LIKE 转义**: 新增 `SafeLike()` 函数，防止 LIKE 通配符注入攻击
- **结账流程重构**: `checkout.asp` 从 1827 行单体文件拆分为 67 行编排器 + 9 个 SSI 子模块
- **安全响应头**: 统一迁移至 `config.asp` 管理，避免 `web.config` 重复设置
- **PWA 支持**: Service Worker 注册、离线缓存策略

### 2.2 V14.6 新增内容

V14.6 在 V14.5 基础上完成全面测试验证与交付优化：

- **整站回归测试**: 50 个后台页面 + 16 个前台页面全部通过 200 OK 验证
- **4 个 P0 阻断性 Bug 修复**: SafeCreatePaymentOrder 重定义、product_settings 条件链断裂、purchase_orders 连接重复包含、stray End If 隐藏管理标签
- **2 个 P1 功能缺陷修复**: amphtmlLink 未初始化变量、purchase_order_post_handler 未声明变量
- **3 个 P2 体验优化**: 3 处内联脚本添加 CSP nonce 支持
- **V14.5 完整备份**: 574 个源文件完整备份至 `网站二_V14.5_backup/`
- **版本信息全面更新**: 全站 V14.5 版本引用统一更新为 V14.6

## 三、前台用户操作指南

### 3.1 用户购物完整流程

```
首页 → 产品列表 → 产品详情 → 定制调香 → 购物车 → 结算 → 支付 → 订单确认
```

**浏览器购物路径**:

1. **首页** (`/index.asp`) — 展示推荐产品、热门分类、促销活动
2. **产品列表** (`/products.asp`) — 支持分类筛选、关键词搜索、排序
3. **产品详情** (`/product.asp`) — 查看香调成分、配料表、价格、用户评价
4. **定制调香** (`/customize.asp`) — 选择香型组合，个性化调配专属香水
5. **配料订购** (`/order_ingredients.asp`) — 单独购买调香原料
6. **购物车** (`/cart.asp`) — 修改数量、删除商品、查看小计
7. **结算** (`/checkout.asp`) — 确认地址、选择支付方式、应用优惠券
8. **支付回调** (`/payment_callback.asp`) — 支付结果处理
9. **订单成功** (`/order_success.asp`) — 订单确认页

## 六、安全特性

### 6.1 身份认证

- **前台用户**: Session 认证，密码使用 SHA256 + Pepper 盐值哈希存储
- **后台管理员**: 独立 Session 认证，支持角色权限分级（管理员/经理/操作员）
- **登录保护**: 失败次数限制、IP 黑名单、登录监控
- **CSRF 防护**: 所有 POST 请求需携带 CSRF Token（32 位随机字符串）

### 6.2 内容安全策略 (CSP)

- 每次请求生成唯一 32 位随机 nonce
- 内联脚本需添加 `nonce="<%= Session("csp_nonce") %>"` 属性
- 当前策略保留 `'unsafe-inline'` 兼容旧代码，后续版本将完全移除

### 6.3 SQL 注入防护

- **参数化查询**: `ExecuteParameterizedQuery()` 使用 ADODB.Command 对象
- **LIKE 转义**: `SafeLike()` 函数转义 `[`, `%`, `_` 通配符
- **输入清理**: 所有用户输入经过 `Replace(str, "'", "''")` 处理

### 6.4 安全响应头

| 响应头 | 值 | 说明 |
|--------|-----|------|
| X-Content-Type-Options | nosniff | 禁止 MIME 类型嗅探 |
| X-Frame-Options | SAMEORIGIN | 禁止跨域 iframe 嵌入 |
| X-XSS-Protection | 1; mode=block | 启用 XSS 过滤器 |
| Referrer-Policy | strict-origin-when-cross-origin | 限制 Referer 信息 |
| HSTS | max-age=31536000; includeSubDomains | 强制 HTTPS |
| Permissions-Policy | camera=(), microphone=(), geolocation=() | 禁用敏感 API |

## 八、部署与运维

### 8.1 环境要求

| 组件 | 要求 |
|------|------|
| 操作系统 | Windows Server 2016+ / Windows 10+ |
| Web 服务器 | IIS 10+ |
| 数据库 | SQL Server 2017+ |
| ASP 引擎 | Classic ASP (VBScript) |
| 代码页 | UTF-8 (CodePage 65001) |

### 8.2 部署检查清单

1. 运行 `/setup/env_check.asp` 检查 IIS 和 SQL Server 版本
2. 运行 `/setup/verify_permissions.asp` 检查数据库权限
3. 运行 `/setup/db_setup.asp` 初始化数据库表结构
4. 确认 `web.config` 中 `setup` 目录已限制访问
5. 确认 SQL Server 允许 Windows 身份验证

### 8.3 数据库备份

- **手动备份**: 访问 `/admin/system/backup_center.asp`
- **自动备份**: 使用 `database/setup_scheduled_backup.ps1` 配置 Windows 计划任务
- **备份脚本**: `database/backup_database.ps1` 支持完整备份和仅代码备份

### 8.4 版本升级流程

1. 备份当前版本: 复制整个站点目录到 `网站二_V{旧版本}_backup/`
2. 应用数据库迁移脚本: 执行 `database/` 目录下的 SQL 升级脚本
3. 更新 `includes/config.asp` 中的 `SYS_VERSION` 和 `SYS_VERSION_NAME`
4. 运行 `setup/env_check.asp` 验证环境
5. 执行全站回归测试

| 问题 | 原因 | 解决方法 |
|------|------|----------|
| HTTP 500 "Name redefined" | SSI 包含重复的 Dim/Function 定义 | 检查 includes 链，避免重复包含 connection.asp |
| HTTP 500 "Missing End" | If/End If 或 Sub/End Sub 不平衡 | 检查跨 SSI 文件的代码块配对 |
| 中文乱码 | 代码页未正确设置 | 确认 `<%@ CodePage=65001 %>` 和 `Response.Charset="UTF-8"` |
| 数据库连接失败 | SQL Server 服务未启动或 Windows 认证失败 | 检查 SQL Server 服务状态和 IIS 应用池标识 |
| CSP 阻止内联脚本 | 脚本缺少 nonce 属性 | 添加 `nonce="<%= Session("csp_nonce") %>"` |
