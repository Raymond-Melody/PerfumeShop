一、系统概述与架构

香氛定制电商系统是一个专注于个性化香水定制的电子商务平台。系统基于 ASP Classic (VBScript) 技术栈构建，运行于 Microsoft IIS Web 服务器上，使用 Microsoft Access JET 4.0 数据库。

系统分为前台用户端和后台管理端两大部分。前台面向普通消费者，提供商品浏览、个性化定制、购物车、下单支付等功能；后台面向运营管理人员，提供订单/商品/客户/生产/采购/财务/系统设置等全方位管理功能。

二、前端页面功能详解

2.1 首页 (index.asp)

首页是全站的门户，包含以下功能模块：

🎠 轮播横幅

3张横幅自动轮播（5秒间隔），宣传"定制专属香氛"、"新品上市"、"满299免运费"等核心卖点

✨ 特色服务

个性定制、天然原料、快速配送、无忧退换 4 大服务承诺展示

📦 产品栏目

根据 ProductTypeConfig 动态展示各类型产品（Custom/Fixed/KOL），每个栏目最多8件商品

📋 定制流程

4步定制流程说明：选择基调 → 搭配香调 → 选择瓶型 → 专属标签

🌸 香调分类

花香调、东方调、木质调、海洋调 4 大分类快捷入口

💬 客户评价

静态客户评价展示区

2.2 商品列表 (products.asp)

全站商品浏览与搜索页面：

  • 筛选：按产品类型（Custom/Fixed/KOL）和价格区间筛选
  • 搜索：支持关键字搜索（商品名和描述）
  • 排序：最新上架、价格从低到高、价格从高到低、名称排序
  • 分页：每页 12 件，支持跳转
  • 商品展示：图片、名称、描述、标签（固定品牌/KOL推荐/奢华）、起价

2.3 商品详情 (product.asp)

商品详情页是定制入口，根据 ProductType 展示不同交互：

  • Custom 类型（定制）：完整的三层香调选择（前调/中调/后调）、容量选择、瓶身选择、刻字标签
  • Fixed 类型（固定品牌）：规格选择（容量+价格的组合）
  • KOL 类型（达人推荐）：类似定制流程但可能限定香调选择范围
  • 公共功能：收藏、加入购物车、查看详细描述

2.4 购物车 (cart.asp)

购物车支持匿名（SessionID）和登录用户（UserID）两种模式：

  • 商品信息：显示每个商品的香调搭配详情（前调/中调/后调成分及比例）、容量、瓶身、刻字
  • 数量调整：支持增减数量、删除单个、清空购物车
  • 选择结算：支持勾选部分商品进行结算
  • 费用计算：商品金额 + 刻字费用 + 运费（满299免运费）
  • 优惠码：预留优惠码输入框（功能尚未对接）
  • AJAX 操作：增删改查均通过 /api/ 目录的 AJAX 接口实现

2.5 结账流程 (checkout.asp)

结账页面是一个综合性表单（约 1700 行代码），功能非常完备：

  • 地址管理：新增/编辑/选择收货地址（省/市/区三级联动，数据来自 /js/area_data.js）
  • 支付方式：支付宝、微信支付、PayPal、货到付款（支付开关在 SiteSettings 表中配置）
  • 配送方式：标准配送、加急配送
  • 发票信息：支持抬头类型（企业/个人）、纳税人识别号
  • 订单备注：可添加客户留言
  • 订单确认：商品清单、费用明细、提交订单
  • 已存在订单：支持从"立即支付"跳转到未支付订单的结算
  • 成分解析：内置 SplitIngredientsUniversal 函数处理定制香水的成分文本

2.6 定制专区 (customize.asp)

定制专区是介绍性页面，但也包含了完整的产品浏览功能：

  • 4步定制流程说明
  • 所有启用的基础香型产品展示（从 Products 表读取）
  • 香调层次说明（前调/中调/后调的详细解释+可用香调列表）
  • 瓶身展示（从 BottleStyles 表读取）
  • 常见问题 FAQ（可折叠展开）

3.2 生产管理 (Production)

生产管理采用 5阶段流水线模型：

1

待排产 (Stage 1)

新订单进入生产队列，调度员安排生产计划。支持优先级标记（高/中/低）和预计完成日期

2

生产中 (Stage 2)

车间按照配方工艺单进行香水调配。涉及 Accord 配比、基香称量、香精混合等工序

3

质检中 (Stage 3)

QC 质量检测，检查香水质检记录（感官评估、密度、色泽等）

4

入库待发 (Stage 4)

成品入库，等待物流发货。库存记录更新（ProductInventory）

5

已出库 (Stage 5)

物流发货，更新快递单号和物流状态

3.7 权限体系（RBAC）

四、安全特性与防护措施

系统在 V7 版本中进行了全面的安全加固，以下是完整的防护体系：

4.1 密码安全

机制实现位置
V2 增强哈希10轮迭代 + Pepper + 非线性混合includes/password_utils.asp
V1/V2 兼容登录时自动检测版本，V1 匹配后自动升级到 V2AdminVerifyAndUpgrade()
记住我 CookieDJB2 签名 + 时间戳 + 30天过期GenerateSecureToken()
密码重置令牌32位随机字符，1小时过期forgot_password.asp

4.2 请求防护

防护类型实现方式
CSRF 防护Session 存储 32 位随机令牌，支持 POST 表单/QueryString/HTTP Header 三种验证方式。jQuery AJAX 自动注入令牌。
SQL 注入SafeSQL() 函数转义单引号；数字参数使用 CLng() 强制类型转换
XSS 防护HTMLEncode() + SafeOutput() 双保险；双引号和单引号都转义为实体
登录速率限制Session 记录失败次数，5 次失败后锁定 15 分钟（管理员和用户独立计数）

4.3 Web.config 安全策略

<!-- database/目录保护：禁止下载 MDB 文件 -->
<location path="database">
  <fileExtensions>
    <add fileExtension=".mdb" denied="true" />
    <add fileExtension=".sql" denied="true" />
  </fileExtensions>
</location>

<!-- includes/目录保护：禁止直接访问 ASP -->
<location path="includes">
  <fileExtensions>
    <add fileExtension=".asp" denied="true" />
  </fileExtensions>
</location>

<!-- 安全响应头 -->
X-Content-Type-Options: nosniff
X-Frame-Options: SAMEORIGIN
X-XSS-Protection: 1; mode=block
Referrer-Policy: strict-origin-when-cross-origin

<!-- 禁用危险 HTTP 动词 -->
OPTIONS, TRACE, DELETE, PUT 均被禁用

<!-- 上传大小限制 -->
maxAllowedContentLength="10485760" (10MB)

4.4 文件上传安全

  • MultipartParser 类：纯 VBScript 实现的 multipart/form-data 解析器，无外部依赖
  • 文件类型验证：扩展名白名单（.jpg/.jpeg/.png/.gif/.webp/.svg）
  • 文件头验证：检测文件头魔数（Magic Bytes），防止伪装扩展名
  • 文件名清洗：SanitizeFileName 移除路径遍历字符（.. / \ : * ? " < > |）
  • 大小限制：最大 5MB（MAX_UPLOAD_SIZE）

4.5 错误处理

  • 所有数据库操作使用 On Error Resume Next 捕获异常
  • 错误信息记录到 Session("LastDBError") 而非直接输出
  • 前端错误信息经过 HTMLEncode 处理后再显示
  • 自定义错误页面配置（web.config 中指定 error.asp）

4.6 审计日志

所有管理员的关键操作均记录到 AdminLogs 表：

INSERT INTO AdminLogs (AdminID, ActionType, ModuleCode, TableName, RecordID, Notes, CreatedAt)
VALUES (adminId, '操作类型', '模块', '表名', '记录ID', '备注', NOW())

记录内容：谁（AdminID）+ 什么操作（ActionType）+ 在哪个模块（ModuleCode）+ 操作了什么数据（TableName+RecordID）+ 何时（CreatedAt）

7.2 常见错误及解决方法

错误 1：数据库无法连接

错误信息：数据库连接失败: 未找到提供程序。该程序可能未正确安装

原因：IIS 应用程序池未启用 32 位应用程序，或 Access 数据库引擎未安装

解决方法：

  1. 打开 IIS 管理器 → 应用程序池 → 选择站点池 → 高级设置
  2. 将"启用 32 位应用程序"设为 True
  3. 如仍不行，安装 Microsoft Access Database Engine 2010 Redistributable

错误 2：页面显示空白

原因：ASP 错误被抑制（On Error Resume Next），错误信息被记录到 Session 但未正确输出

解决方法：

  1. 在 web.config 中设置 <customErrors mode="Off" /> 临时查看详细错误
  2. 检查 Session("LastDBError") 的内容
  3. 检查 IIS 中 ASP 的"启用父路径"是否设为 True

错误 3：800a0bb9 参数类型不正确

错误信息：执行 SQL 时出现 800a0bb9 参数类型不正确

原因：在 Access 中错误地使用了 conn.RecordsAffected 属性

解决方法：将 conn.Execute(sql) 改为 conn.Execute sql, recordsAffected, 128（128 = adExecuteNoRecords）

错误 4：80040e37 找不到表

错误信息：Microsoft JET Database Engine 错误 '80040e37' 找不到表

原因：SQL 中引用了不存在的表名。Access 中表名是 OrderDetails 不是 OrderItems

解决方法：检查 SQL 语句中的表名，使用正确的 Access 表名

错误 5：800a01a8 缺少对象（VBScript）

原因：VBScript 的 And 操作符不对表达式求值做短路处理。如 If Not rs Is Nothing And Not rs.EOF Then，当 rs 为 Nothing 时仍会求值 rs.EOF

解决方法：拆分为嵌套 If 或使用 If rs Is Nothing Then ... ElseIf rs.EOF Then ... Else ... End If

错误 6：上传文件失败 (ASP 0104)

错误信息：ASP 0104 不允许操作

原因：ASP 默认最大上传大小为 200KB，超过此限制需要配置

解决方法：在 web.config 中已设置 maxAllowedContentLength="10485760"（10MB），还需在 IIS 管理器中修改 ASP 限制：
IIS → ASP → 限制属性 → 最大请求实体主体限制 → 设为 10485760

错误 7：VBScript 运行时错误 '800a000d'

原因：类型不匹配，通常发生在将字符串当作数字运算时

解决方法：使用 IsNumeric() 检查后再转换，或使用 CDbl("0" & val) 安全转换

7.3 部署指南

全新部署步骤

  1. 安装 IIS：控制面板 → 程序和功能 → 启用或关闭 Windows 功能 → Internet Information Services
  2. 启用 ASP 功能：IIS → 世界范围 Web 服务 → 应用程序开发功能 → ASP（勾选所有子项）
  3. 启用 32 位应用程序：IIS 管理器 → 应用程序池 → DefaultAppPool → 高级设置 → 启用 32 位应用程序 = True
  4. 复制网站文件：将整个备份包 网站二_clean 复制到目标路径
  5. 设置权限：
    • database\ 目录：给 IIS_IUSRS 读取/写入权限（Access 数据库需要写入权限）
    • images\products\、images\uploads\ 等：给 IIS_IUSRS 写入权限
  6. 创建 IIS 站点：IIS 管理器 → 添加网站 → 指定名称、物理路径、端口（如 80 或 8080）
  7. 测试访问：打开浏览器访问 http://localhost/index.asp

从备份恢复

  1. 将 网站二_clean 目录完整复制到目标服务器
  2. 确保 web.config 文件存在（包含安全策略配置）
  3. 如更换域名，修改 includes/config.asp 中的 SITE_URL 常量
  4. 如需修改数据库密码，更新 includes/connection.asp 中的连接字符串
  5. 建议修改 includes/password_utils.asp 中的 PASSWORD_PEPPER 常量
