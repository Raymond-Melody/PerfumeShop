### P4 — 渐进式架构现代化 (Tasks 21-24)

#### Task 21: API 认证与限流（ASP 替代 Node.js 网关）
- **`includes/api_auth.asp`** *(新建)*:
  - `Function API_AuthCheck()` — 统一 Boolean 认证函数
    - Mode 1: Session 认证（已登录用户自动通过）
    - Mode 2: API Key + HMAC-SHA256 签名验证（支持 MD5 回退）
    - 时间戳防重放攻击 (±300s 窗口)
  - `API_GenerateSignature()` — HMAC 签名生成辅助函数
- **`includes/rate_limiter.asp`** *(新建)*:
  - 基于 `Application("RL_Buckets")` Dictionary 的令牌桶算法
  - `RL_Check(key, maxRequests, windowSec)` — 令牌桶检查
  - `RL_GetRemaining(key)` — 剩余令牌查询
  - `RL_Cleanup()` — 自动清理过期桶
  - `RateLimitCheck(bucketName)` / `RateLimitSend429(bucketName)` — api_guard 兼容包装
  - 429 Too Many Requests 标准响应（Retry-After 头）
- **`includes/api_guard.asp`** *(新建)*:
  - `API_Guard(bucketName, requireAuth)` — 统一 API 前置守卫
  - 先速率限制 → 再认证检查，失败自动发送 JSON 错误响应
  - 已集成到 12 个 API 端点

#### Task 22: 前端组件化改造（Web Components）
- **`js/components/product-card.js`** *(新建)*:
  - `<product-card>` 自定义元素，Shadow DOM 封装
  - 属性：`data-id`、`data-name`、`data-price`、`data-image`、`data-category`、`data-rating`、`data-badge`、`data-link`、`data-lazy`
  - IntersectionObserver 懒加载
  - Hover 遮罩层 (名称 + 评分 + 价格 + 查看详情按钮)
  - 类型徽章（定制/限定/新品）
- **`js/components/review-stars.js`** *(新建)*:
  - `<review-stars>` 自定义元素
  - 只读模式 (`data-readonly`) 和交互模式
  - 交互模式：点击评分、Hover 预览、Alt+Click 半星选择
  - 触发 `change` CustomEvent `{ value: number }`
  - 自动创建隐藏 `<input>` 用于表单提交
- **`js/components/search-autocomplete.js`** *(新建)*:
  - `<search-autocomplete>` 自定义元素
  - 350ms debounce 输入 + `AbortController` 请求取消
  - 键盘导航（↑ ↓ 移动、Enter 选择、Esc 关闭）
  - `data-api` 端点配置（默认 `/api/search_suggestions.asp`）
  - 底部 "查看全部结果" 链接
- **`includes/footer.asp`** (增强): 添加 3 个 Web Component 脚本引用（`defer` 加载）
- **`product.asp`** (增强): 集成 `<review-stars>` 组件替代传统星级选择器
- **`index.asp`** (增强): 集成 `<product-card>` 和 `<search-autocomplete>` 组件

#### Task 23: 数据库性能优化
- **`database/v18_perf_indexes.sql`** *(新建)*:
  - 13 个性能索引，包含幂等性检查 (`IF NOT EXISTS`)
  - Products: `ProductType+IsActive`、`Category+IsActive`、`CreatedAt+IsActive`、KOL 过滤、`ProductName` 全文
  - Orders: `UserID+CreatedAt`、`Status+CreatedAt`
  - OrderItems: `OrderID+ProductID` 复合索引
  - AppLogs: `CreatedAt+LogType`、`LogLevel` 过滤索引
  - UserBehavior: `UserID+BehaviorType+CreatedAt`
  - 全部使用 `WITH (ONLINE=ON, SORT_IN_TEMPDB=ON, FILLFACTOR=90)`
  - 附带 `UPDATE STATISTICS ... WITH FULLSCAN` + 索引状态查询
- **`includes/cache_v18_ext.asp`** *(新建)*:
  - `CM_GetOrEmpty(key)` — 简化缓存读取，无缓存返回 Empty
  - `CM_CacheProductQuery(cacheKey, ttlSeconds)` — 产品查询缓存包装
  - `CM_ClearProductCaches()` — 一键清除所有产品相关缓存
  - `CM_RecordCacheHeaders(isHit)` — 添加 `X-Cache: HIT/MISS` 响应头
  - `CM_GetCacheStatsJSON()` — JSON 格式缓存统计
- **`includes/cache_manager.asp`** (增强): 引入 `cache_v18_ext.asp`

#### Task 24: 监控与可观测性
- **`includes/metrics.asp`** *(新建)*:
  - `METRICS_RecordPageLoad(pageName, startTime, userId)` — 页面加载耗时记录，超阈值自动写入 AppLogs
  - `METRICS_RecordApiTime(apiName, startTime)` — API 响应时间记录
  - `METRICS_TrackFunnel(step, userId, targetId)` — 购物车转化漏斗埋点 (view_product → add_to_cart → begin_checkout → purchase)
  - `METRICS_GetFunnelStats()` — 7天漏斗统计
  - `METRICS_GetSummary()` — JSON 格式指标摘要
- **`api/health_check.asp`** (增强):
  - 新增内存使用率检测（Application 变量统计）
  - 慢查询统计（读取 DAL 慢查询计数器）
  - 缓存命中率展示

## 部署说明

### 前提条件
1. IIS + Classic ASP 运行环境
2. SQL Server (本地 MSSQL17.YOURPERFUME 实例)
3. MSOLEDBSQL 驱动已安装并配置 TCP/IP 协议

### 升级步骤
1. **数据库升级**: 按顺序执行 `database/` 下的 V18 SQL 脚本：
   - `v18_member_tiers.sql` → `v18_points_system.sql` → `v18_coupon_system.sql`
   - `v18_flash_group_activities.sql` → `v18_subscription.sql` → `v18_community_ugc.sql`
   - `v18_perf_indexes.sql`（最后执行，幂等安全）
2. **代码部署**: 将所有 ASP/JS/CSS 文件覆盖到网站根目录
3. **Feature Flags**: 确认 `includes/config.asp` 中所有 V18 Feature Flags 状态正确
4. **缓存清理**: 重启 IIS 或回收 Application Pool
5. **冒烟测试**: 运行 `powershell -ExecutionPolicy Bypass -File test_v18_smoke.ps1`

### 回滚方案
- 关闭 Feature Flags: 将 `config.asp` 中所有 V18 新增 Flags 设为 `False`
- 数据库脚本包含幂等性检查，不会重复执行
- 性能索引可通过 `DROP INDEX` 回滚

## 升级注意事项

1. **API 认证默认开启**: 外部 API 调用需携带 API Key + HMAC 签名，或通过 Session 认证
2. **速率限制默认 60req/60s**: 可在 `rate_limiter.asp` 中调整 `RL_DEFAULT_MAX` 和 `RL_DEFAULT_WINDOW`
3. **Web Components 需现代浏览器**: Chrome 67+, Firefox 63+, Safari 10.1+, Edge 79+
4. **性能索引**: `ONLINE=ON` 要求 SQL Server Enterprise Edition，Standard Edition 需去掉此选项
5. **DAL 规范**: 所有新数据库查询必须使用 DAL 参数化函数，禁止字符串拼接 SQL
6. **CSP Nonce**: 内联脚本必须使用 `nonce="<%= Session("csp_nonce") %>"`
